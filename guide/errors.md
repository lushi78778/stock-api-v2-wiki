# 错误码说明

失败时 HTTP 状态码与 JSON 中的 `code` 会一起告诉你问题类型。排障时务必记录响应里的 **`requestId`**。

| code | HTTP | 默认 message | 常见原因 |
|------|------|--------------|----------|
| 40001 | 400 | 参数非法 | 日期格式错误、缺少必填参数、symbols 为空 |
| 40101 | 401 | 未登录或登录已失效 | Token 缺失、错误、已禁用或过期 |
| 40301 | 403 | 无权限访问 | Token scope 不足 |
| 40401 | 404 | 资源不存在 | 证券代码不存在、区间无快照数据 |
| 40901 | 409 | 状态冲突 | 门户侧业务冲突（开放行情较少见） |
| 42901 | 429 | 请求已超出额度限制 | 触发限流，请降频 |
| 50001 | 500 | 服务内部错误 | 服务端异常 |
| 50301 | 503 | 服务暂时不可用 | 只读行情库未就绪 |

## 错误响应示例

Token 未携带或错误时：

```json
{
  "code": 40101,
  "message": "未登录或登录已失效",
  "data": null,
  "requestId": "req_xxx",
  "ts": "2026-05-17T20:00:00"
}
```

参数错误时：

```json
{
  "code": 40001,
  "message": "参数非法",
  "data": null,
  "requestId": "req_xxx",
  "ts": "2026-05-17T20:00:00"
}
```

你可以这样理解：先看 HTTP 状态，再看 `code` 数字；`message` 是人类可读提示，具体字段问题以返回文案为准。
